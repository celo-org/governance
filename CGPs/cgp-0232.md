## Overview

This governance proposal resets the internal accounting of the `CeloUnreleasedTreasury` contract and transfers 150,000,000 CELO to a designated address. It uses OP Stack's `StorageSetter` contract to safely modify proxy storage, combined with a temporary registry swap to authorize the transfer.

## Background

The `CeloUnreleasedTreasury` contract uses an internal `hasAlreadyReleased` flag and a `remainingBalanceToRelease` counter to track released funds. On the first call to `release()`, the contract snapshots `address(this).balance` into `remainingBalanceToRelease` and sets `hasAlreadyReleased = true`. All subsequent releases decrement from this snapshotted value.

The `release()` function is restricted to the `EpochManager` via the `onlyEpochManager` modifier. Since governance cannot directly call `release()`, a temporary registry swap is required.

### Storage Layout (CeloUnreleasedTreasury)

| Slot | Offset | Variable                    | Type    |
| ---- | ------ | --------------------------- | ------- |
| 0    | 0      | `_owner`                    | address |
| 1    | 0      | `registry`                  | address |
| 2    | 0      | `_status`                   | uint256 |
| 3    | 0      | `initialized`               | bool    |
| 3    | 1      | `hasAlreadyReleased`        | bool    |
| 4    | 0      | `remainingBalanceToRelease` | uint256 |

### Prior Art

CGP-0207 attempted a similar registry swap to call `release()` but was rejected. This proposal improves on that approach by also resetting `hasAlreadyReleased` via `StorageSetter` to ensure the contract re-snapshots the correct balance after the transfer.

## Proposed Changes

This proposal executes 5 transactions atomically:

### Transaction 1: Reset `hasAlreadyReleased` via StorageSetter

- **Destination**: `CeloUnreleasedTreasuryProxy._setAndInitializeImplementation`
- **Data**: Temporarily points the proxy to `StorageSetter` and calls `setBytes32(slot=3, value=0x01)` which sets `initialized=true, hasAlreadyReleased=false`
- **Value**: 0
- **Effect**: Resets the accounting flag so the next `release()` call re-snapshots `address(this).balance`

### Transaction 2: Restore original implementation

- **Destination**: `CeloUnreleasedTreasuryProxy._setImplementation`
- **Data**: Restores the CeloUnreleasedTreasury implementation at `0x0148a25D13cBD39D2FF5749369356CbdE20ec9d7`
- **Value**: 0

### Transaction 3: Temporarily set Governance as EpochManager

- **Destination**: `Registry.setAddressFor`
- **Data**: Sets `EpochManager` registry entry to the Governance contract (`0xD533Ca259b330c7A88f74E000a3FaEa2d63B7972`)
- **Value**: 0
- **Effect**: Allows the Governance contract (executing this proposal) to pass the `onlyEpochManager` check

### Transaction 4: Transfer 150,000,000 CELO

- **Destination**: `CeloUnreleasedTreasury.release`
- **Data**: Calls `release(0x000000000000000000000000000000000000dEaD, 150000000000000000000000000)`
- **Value**: 0
- **Effect**: Since `hasAlreadyReleased` was reset, this re-snapshots `remainingBalanceToRelease` from actual balance, then transfers 150M CELO

> **Note**: The recipient address `0x...dEaD` is a placeholder. Replace with the actual destination before submission.

### Transaction 5: Restore EpochManager in Registry

- **Destination**: `Registry.setAddressFor`
- **Data**: Restores `EpochManager` to `0xF424B5e85B290b66aC20f8A9EAB75E25a526725E`
- **Value**: 0

## Risks

- **Temporary StorageSetter exposure**: Between transactions 1 and 2, the proxy briefly points to `StorageSetter`. Since all transactions execute atomically within a single governance execution, there is no window for external interference.
- **Registry swap window**: Between transactions 3 and 5, the Governance contract is registered as `EpochManager`. Any contract querying the registry for `EpochManager` during this window would get the Governance address. Again, atomic execution prevents external exploitation.
- **Balance sufficiency**: The treasury currently holds ~287.7M CELO, and `remainingBalanceToRelease` matches. The 150M transfer is well within bounds.
- **Post-execution accounting**: After execution, `remainingBalanceToRelease` will equal the actual balance minus 150M (~137.7M CELO). Future `release()` calls by EpochManager will work correctly.
