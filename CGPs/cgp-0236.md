## Summary

This proposal implements one of the four changes outlined in the [CELOccelerate tokenomics temperature check (CGP-0233)](cgp-0233.md), which recently passed governance. Specifically, it executes item 4: pausing carbon offset fund contributions.

It pauses all carbon fund payments across both payment streams in the Celo protocol:

1. **EpochRewards** — epoch-based carbon offsetting rewards (currently 0.05% of target supply increase)
2. **FeeHandler** — fee-based carbon fund distributions (currently 10% of transaction fees)

Both are paused by setting their respective carbon fractions to zero via governance.

## Onchain Changes

### Transaction 1: Pause EpochRewards Carbon Fund

- **Destination**: `EpochRewards` at `0x07F007d389883622Ef8D4d347b3f78007f28d8b7`
- **Function**: `setCarbonOffsettingFund(address,uint256)`
- **Args**: `["0xCe10d577295d34782815919843a3a4ef70Dc33ce", "0"]`
- **Value**: 0

Sets the `carbonOffsettingFraction` to 0. The partner address is kept unchanged so this change is easily reversible. The epoch reward calculation at `calculateTargetEpochRewards()` will return 0 for the carbon fund component, and the `CeloUnreleasedTreasury.release()` call in `EpochManager` will transfer 0 CELO.

### Transaction 2: Pause FeeHandler Carbon Fund

- **Destination**: `FeeHandler` at `0xcD437749E43A154C07F3553504c68fBfD56B8778`
- **Function**: `setCarbonFraction(uint256)`
- **Args**: `["0"]`
- **Value**: 0

Sets the carbon fraction to 0. The freed fraction is redirected to the burn (since `burnFraction = 1 - carbonFraction - otherBeneficiariesFraction`). There is one other beneficiary on mainnet (`0x7A1E98FC9a008107DbD1f430a05Ace8cf6f3FE19`), so `totalFractionOfOtherBeneficiariesAndCarbon` remains non-zero and no division-by-zero can occur in `_distribute()`.

## Risks

- **Reversibility**: This change is fully reversible. A future governance proposal can restore the carbon fractions to their previous values.
- **Economic impact**: The CELO that would have been directed to the carbon fund will instead remain unminted (EpochRewards) or be burned (FeeHandler). This has a mildly deflationary effect.
- **No smart contract risk**: Setting fractions to zero is a supported code path. ERC-20 transfers of 0 are valid, and the FeeHandler has an existing other beneficiary preventing any division-by-zero in `_distribute()`.
